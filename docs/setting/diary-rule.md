# Diary Rule

作業記録は `docs/diary/YYYYMMDD.md` に残す。

## 目的

- その日に何を決めたかを後から追えるようにする。
- 変更した主要ファイルと理由を残す。
- 未決事項や次の作業を見失わないようにする。

## 書式

```markdown
# YYYY-MM-DD 作業記録

## 1. 概要

その日の作業全体の短い要約。

## 2. 目的

なぜ作業したか。

## 3. 作ったもの

- [役割](相対パス)
  - 何を作ったか。

## 4. 変更した主要ファイル

- [ファイル名](相対パス)
  - 変更内容。

## 5. 検証結果

実行した確認や、未実行ならその理由。

## 6. 注意点

運用上の注意、リスク、判断理由。

## 7. 残タスク

- 未完了の作業。
```

## ルール

- 同じ日のファイルが既にある場合は、上書きではなく追記または整理更新する。
- secret、トークン、private URLは書かない。
- ファイルへの参照は相対パスで書く。
- 長すぎるログではなく、あとから判断できる粒度に要約する。
