# Debate Protocol

作成日: 2026-05-24
更新日: 2026-05-24

## 目的

MoChat 上での設計書討論を、再現可能な手順で実行するための運用プロトコルを定義する。

## 対象

- `src/debate/orchestrator.py`
- `src/debate/protocol.py`
- `src/debate/convergence.py`
- `src/debate/monitor.py`

## ラウンド進行

1. セッション準備
- `sessions/create` で討論セッションを作成する。
- 対象設計書パス、議題、終了条件をメタデータとして保持する。

2. ラウンド開始（`round = 1..max_rounds`）
- 発言順は `skiller -> hawk -> keeper -> mediator` を固定とする。
- 各エージェントは最新履歴と対象設計書を入力にして 1 回発言する。

3. メドラー判定
- メドラーは各ラウンド末尾で必ず `合意/対立/保留` のいずれかを返す。
- 返却形式は次の 4 要素を必須とする。
  - `合意点`
  - `対立点`
  - `保留理由（必要時）`
  - `次ラウンド焦点`

4. 収束判定
- `合意`: 討論終了。結果保存へ進む。
- `対立`: 次ラウンドへ進む。
- `保留`: 討論終了し、人間判断待ちへ移行する。
- `max_rounds` 到達時: 自動的に「保留」扱いで終了する。

## 停止条件

- `max_rounds` 到達
- メドラーが `合意` を返却
- メドラーが `保留` を返却
- 監視ロジックが強制停止を要求（品質低下や逸脱が閾値超過）

## 例外時の扱い

- MoChat API エラー: 1 回目は指数バックオフで再試行、連続失敗でラウンド中断。
- LLM タイムアウト: 該当エージェント発言を `timeout` と記録し、メドラーに継続可否を委譲。
- 不正フォーマット: メドラー出力を `convergence.py` で正規化し、足りない項目は `unknown` 扱い。

## 成果物出力

- 終了時に `docs/debate-results/YYYY-MM-DD_<topic-slug>.md` を保存する。
- 保存フォーマットは `docs/debate-results/template.md` を使う。
